$ rc --help
rootcause CLI — a thin, scriptable client over the rootcause API

Usage:
  rc [command]

Start:
  status      Health summary + recent runs
  ask         Trigger a run from a question and wait for the answer
  run         Inspect and manage the run lifecycle

Manage:
  project     Manage project configuration and resources

Develop:
  dev         Develop and inspect project behavior

Operate:
  fleet       Operate and inspect project health
  admin       Box-level administration (users/projects/catalog; global-admin token)

Local:
  auth        Manage local authentication and inspect access
  self        Manage the rc installation and shell integration

Flags:
      --no-preview       write large output artifacts but omit head/tail previews
      --out-dir string   directory for large output artifacts (default: RC_OUTPUT_DIR or .rootcause/output)
  -o, --output string    output format: json|table (default: auto-detect)
      --profile string   token profile to use (default: auto — the brain in the current directory, else "default")
      --project string   scope the request to one project by name or id (requires an all-projects token)
      --raw-output       disable progressive output spill and print full payloads to stdout
      --scope string     force request routing: project|tenant. "project" clears any resolved tenant (--tenant, a brain checkout, or a tenant-bound login) so a tenant-capable command hits the project route; "tenant" requires a resolvable tenant. With a tenant-pinned token, --scope project routes to the project and the server returns 403 — the flag controls routing, not authorization.
      --tenant string    override the login tenant where supported

Use "rc [command] --help" for more information about a command.

$ rc ask --help
Trigger a run from a question and wait for the answer
